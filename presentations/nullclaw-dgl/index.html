Номинация · технический вызов

Самая важная правка: default-deny на provider to tool path

Если модель видит [EMAIL_1], runtime не должен автоматически разворачивать это в raw email перед любым tool. Иначе redaction превращается в confused-deputy bypass

Before risk

Модель не знает PII, но может попросить runtime его отправить

provider sees: "call http_request with [EMAIL_1]" old runtime: [EMAIL_1] -> alice@example.com risk: network / shell / file-like tool
